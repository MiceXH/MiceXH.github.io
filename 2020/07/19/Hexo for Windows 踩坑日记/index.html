Hexo for Windows 踩坑日记

前言

记得2015年刚上大学的时候老韩帮我搞了阿里云的ECS、域名、备案…，我有了一个属于自己的WordPress博客。现在5年过去了，老韩在美团和兴哥做兄弟，我则跑回二线做苦逼的出差IT狗。

(PS.有趣的是，当我写下Hexo新博客的第一篇文章时，老韩主页上挂着一篇2018年写的用Hugo搭建博客的教程，我真是跟不上时代了233)。

机缘巧合下，这个博客诞生了，读大学时用Mac，工作后反而因为技术栈改用Windows，这可能是一种莫名的讽刺的吧。多余的话放到另一篇文章说吧，我们进入正题。

零、说明

网络上介绍如何使用Hexo搭建博客的教程很多，本文旨在记录我所遇到的问题，并分享出来。详细教程推荐B站Up主CodeSheep的视频0，这个视频也是促成这个博客的诞生重要原因之一。

似乎极客（或者说硬核程序员）对于视频教程抱有一种鄙视。当然，在一定程度上，视频教程，尤其是“xxx小时带你学Java”一类的培训教程确实引人鄙视，但客观来说视频要比图文来的直观，至于说信息的浓度我觉得并不能一刀切地说图文所包含的信息更多，在一些文字和图片很难描述清楚的情况下，就需要引入视频，GIF对一篇文章解释力的提高相信每一个看力扣题解的人都深有体会。

(PS.有条件的话，我也想试着成为一名Up主，默默立下一个Flag)

一、Node.js配置环境变量

这一步并非有意为之，只是在我困惑需不需要手动为Node.js添加环境变量时搜索到了一篇文章。“Node.js在默认安装完成后，在执行 npm install [模块名] -g命令时，会将模块安装到[C:\Users\xxx\AppData\Roaming\npm]路径下” 。而我贫瘠的C盘只有128GB，所以就照着操作了一遍。我相信原博主的文章更加通俗易懂，就不赘述这一过程了。此外，这篇文章1还介绍了如何切换Node.js至淘宝源。

二、Git配置

说来惭愧，我司莫得版本控制，也莫得代码库。不用说Git，SVN也不用，一个人就是一支队伍。所以平日里我就是个Git小白，上一次用Github还是在大二课程设计时，远远落后于时代。
